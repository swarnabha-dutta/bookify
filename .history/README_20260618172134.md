## Overview

</div>

Bookify is a modern web application designed to make reading more interactive. The long-term product direction is clear in the codebase: upload books, process content with AI, and enable voice-driven conversations around what users read.

As of now, the project has a polished front-end foundation, a working authentication flow, protected routing, reusable UI building blocks, comprehensive file upload infrastructure backed by Vercel Blob, and production-ready API endpoints for PDF ingestion.

## Data Model

</div>

The Bookify application uses a MongoDB-based data model designed to support book uploads, segmentation, and voice interaction sessions. Here's the current schema architecture:

![Book Voice Session Data Model](./diagrams/Bookify%20Schema%20Design.png)

### Schema Overview

**books** - Core book entity
- `id` (ObjectId, primary key) - Unique identifier for each book
- `clerkId` (string) - Owner's Clerk authentication ID
- `title`, `author`, `slug` (string) - Book metadata
- `persona` (string) - AI persona for voice interactions
- `fileURL`, `fileSize` (string/number) - PDF storage reference
- `fileBlobKey` (string) - Blob storage identifier
- `coverURL`, `coverBlobKey` (string) - Cover image storage
- `totalSegments` (number) - Total parsed segments
- `createdAt`, `updatedAt` (timestamp) - Audit timestamps

**book_segments** - Parsed content chunks
- `id` (ObjectId, primary key) - Segment identifier
- `clerkId` (string) - Owner reference
- `content` (string) - Extracted text
- `segmentIndex` (number) - Position in book
- `pageNumber`, `wordCount` (number) - Content metrics
- `bookId` (ObjectId, foreign key) - Reference to parent book
- `createdAt`, `updatedAt` (timestamp) - Metadata

**voice_sessions** - Conversation records
- `id` (ObjectId, primary key) - Session identifier
- `bookId` (ObjectId, foreign key) - Associated book
- `clerkId` (string) - User reference
- `startedAt`, `endedAt` (timestamp) - Session duration
- `durationSeconds` (number) - Total duration
- `billingPeriodStart` (timestamp) - Billing reference
- `createdAt`, `updatedAt` (timestamp) - Tracking

This structure enables efficient book storage, AI-powered segment processing, and voice conversation tracking for the platform's core features.

## API Endpoints

</div>

### File Upload Endpoint

**`POST /api/upload`** - Upload and store media files to Vercel Blob

#### Request
```json
{
  "file": "FormData",
  "body": {
    "type": "blob"
  }
}
```

#### Security & Validation
- **Authentication**: Requires valid Clerk session
- **Supported MIME types**: `application/pdf`, `image/jpeg`, `image/png`, `image/webp`
- **File size limit**: Configurable via `MAX_FILE_SIZE` constant (default: 50MB)
- **Error handling**: Comprehensive error responses with user-friendly messages

#### Response
```json
{
  "blob": {
    "url": "https://blob-storage.vercel-storage.com/...",
    "downloadUrl": "https://blob-storage.vercel-storage.com/...",
    "pathname": "filename.ext",
    "contentType": "application/pdf",
    "contentDisposition": "inline; filename=\"filename.ext\"",
    "size": 12345,
    "uploadedAt": "2026-06-18T14:00:00Z"
  }
}
```

#### Implementation Details
- Built with `@vercel/blob` client SDK
- Enforces user identification via `BLOB_READ_WRITE_TOKEN`
- Token payload includes authenticated user ID for audit trailing
- Automatic file suffix randomization for naming collisions
- Production-grade error handling with graceful fallbacks

## Environment Setup

</div>

### Required Variables

**Authentication & Authorization**
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` - Clerk public key for client-side auth
- `CLERK_SECRET_KEY` - Clerk secret key for server-side operations

**Cloud Storage**
- `BLOB_READ_WRITE_TOKEN` - Vercel Blob API token with read/write permissions

**AI Services** (when implemented)
- `NEXT_PUBLIC_ASSISTANT_ID` - Reference to the AI assistant instance
- `VAPI_PRIVATE_KEY` - Vapi authentication token (optional, for voice features)

**Database** (when implemented)
- `MONGODB_URI` - MongoDB connection string
- `MONGODB_DB` - Database name (optional, defaults to workspace name)

### Optional Variables

- `MAX_FILE_SIZE` - Maximum upload file size in bytes (default: 52MB)
- `NEXT_ENV` - Environment designation (development/production)
